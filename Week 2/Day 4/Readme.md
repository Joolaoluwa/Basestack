# AWS CloudTrail and audit

Cloudtrail is a tool that monitors the accounts and api activities in the cloud, this helps to create a list of activities that are taken place in the cloud. This is useful to document and understand what, who, where, and when taken place in the cloud.

- Who - Who is taken the action based on the user identity
- Where - The source IP taken that is the source of the request
- What - The api call which is made
- When - Timestamp

Cloudtrail event types include management trails for noting management and operation of resources such as creating and deleting buckets it is enabled by default, the data cloud trail that tracks what have done on resources, insights detects unusual activities in the cloud.
